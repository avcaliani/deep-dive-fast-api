Schrute Bucks Rewards

Dunder Mifflin's employee rewards program.

Show endpoints
GET / home page 
curl -s http://127.0.0.1:8000/
POST /auth login 
curl -s -X POST http://127.0.0.1:8000/auth \
  -H "Content-Type: application/json" \
  -d '{"username": "dwight@dundermifflin.com", "password": "Test1234!"}'

Needs a matching user already in MongoDB — run make create-user to create this exact demo user if you haven't yet.

Everything below needs an employee badge — set $TOKEN from the /auth response first: export TOKEN=...

GET /mood/{item} daily check-in 
curl -s http://127.0.0.1:8000/mood/😁 \
  -H "Authorization: Bearer $TOKEN"

item is one of 😁, 😡, 🤔.

GET /vending/ browse catalog 
curl -s http://127.0.0.1:8000/vending/ \
  -H "Authorization: Bearer $TOKEN"
GET /vending/{item} item by index 
curl -s http://127.0.0.1:8000/vending/0 \
  -H "Authorization: Bearer $TOKEN"
POST /vending/{item} redeem 
curl -s -X POST http://127.0.0.1:8000/vending/0 \
  -H "Authorization: Bearer $TOKEN"

Spends Schrute Bucks — returns 402 if your balance can't cover the item's cost.
